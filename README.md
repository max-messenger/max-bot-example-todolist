# Bot TODOLIST

Это демонстративный бот, котрый можно использовать как в формате чат-бота, так и в виде мини-приложения

## 1. Установка и запуск бота

1. Скачайте репозиторий:

```bash
git clone git@github.com:max-messenger/max-bot-example-todolist.git
```

2. Запустите локально docker-compose в корне проекта:

```bash
docker-compose up -d
```

3. Подготовьте конфигурационный файл (скопируйте его в корень проекта):

```bash
cp config/todolist/config.tmpl.yaml .
```

4. Соберите проект (в результтате будет собран исполняемый бинарный файл проекта):

```bash
// Параметр -o определяет название файла
go build -o todo-app ./cmd/todolist
```

Создание приложения происходит через фабрику в файле `app.go`, возращается объект `fx.App`.
Менять ничего не нужно, модули добавляютя в файле `fx.go`

```go
func CreateApp(cfgName string) *fx.App {
    box := gibs.NewBox(
        Modules,
        gibs.WithAppName("todolist"),
        gibs.WithConfigFile(cfgName),
    )

    return box.CreateApp()
}
```

5. Запустите бота:

```bash
// Параметр -c указывает файл
./todo-app -c config.yaml
```

## 2. Работа TODOLIST в чате

Бот работает по принципу to-do - листа. Для нчала работы, откройте чат с ботом, далее вы можете:

- Вызвать список функций бота (функция `/info` в чате)
- Добавлять записи (функция `/create` в чате)
- Просмотривать все добавленные записи (функция `/list` в чате)
- Удалять записи из списка (функция `/delete` в чате)
- Получить свой user_id и chat_id (функция `/myid` в чате)

## 3. Работа TODOLIST в мини-приложении

Для вызова мини-приложения, зайдите в чат с ботом и нажмите **Открыть**, далее вы можете:

- Напечатать текст в поле ввода и нажать **Добавить**. Введенный текст сразу отобразится в списке с открытым чек-боксом (по количеству задач в списке и количеству символов в одной задаче ограничений нет)
- Перевести задачу в статус выполненной, установив чекбокс
- Удалить задачу из списка
- Отфильтровать список по **Выполненным** или **Невыполненным** задачам (по умолчанию задан фильтр **Все**)

В конце списка отображается количество активных задач

Основные гайдлайны по разработке сервисов с использованием todolist можно посмотреть в [guidelines](guidelines/README.md)

## 4. Структура проекта

Если установка прошла успешно, то появится папка `internal/app`, со следующей структурой:

```bash
internal/app
├── clients          - клиенты для работы с внешними сервисами(http, grpc, bots)
│   ├── maxbot       - клиент MAX, подключение к боту и получение/отправка сообщений
│   └── fx.go        - модули clients
├── domain           - доменные модели для общения между сервисами
├── repository
│   ├── staterepo    - репозиторий для хранения состояния пользователя
│   ├── todorepo     - репозиторий для хранения todo
│   └── fx.go        - модули repository
├── router           - HTTP API сервиса
│   ├── todolistctrl - контроллер для todolist
│   └── webhookctrl  - контроллер для webhook нужен для получения сообщений из бота в режиме webhook
├── services
│   ├── analytic     - сервис для отправки аналитики в kafka
│   ├── bot          - сервис для обработки сообщений из бота
│   ├── todolist     - сервис для работы с todo
│   └── fx.go        - модули services
└── subscriber       - подписки на события из бота, сюда можно добавлять свои подписки, например из kafka
```

## 5. Описание модулей проекта

### 5.1 Модули по умолчанию

В проекте используются следующие дефолтные модули:

```go
var defaultModules = fx.Options(
    grace.Module, // модуль для запуска приложения в фоне
    health.Module, // модуль для проверки статуса приложения
    server.Module, // модуль для запуска сервиса в http
    info.Module, // модуль для получения информации о приложении
    telemetry.Module, // модуль для получения метрик в prometheus и otel и профайлинг
)
```

HTTP-сервис для проверки статуса приложения и сбора метрик запускется отдельно (по умолчанию на порту `8082`)

### 5.2 Состав модулей в Package

Набор модулей для работы приложения, http, grpc сервер, сбор трейсов и метрик, логи, http middleware, marshaler

Часть модулей включена по умолчанию в проект, другие модули нужно добавить в файле `internal/app/fx.go`

```go
var Modules = fx.Options(
    //.....
    moduleName.Module, 
    //.....
)
```

Модифицировать код внутри `pkg` не надо, если требуются доработки обращайтесь к мейнтейнеру.

#### [Background Tasker(bgtasker)](pkg/bgtasker/README.md)

Модуль для запуска фоновых задачь в приложении

#### [Grace](pkg/grace/README.md)

Модуль для управления жизненным циклом сервисов с поддержкой graceful shutdown.
Включён в шаблон проекта.

#### [HTTP](pkg/http/README.md)

Модуль для работы с http сервисами, в основном это набор middleware для запуска сервисов в http.

#### [Health](pkg/health/README.md)

Модуль для получения информации о состоянии приложения.
Включён в шаблон проекта, работает на системоном порту `/health`

#### [Info](pkg/info/README.md)

Модуль для получения информации о приложении через HTTP endpoint.
Включён в шаблон проекта, работает на системоном порту `/info`

#### [Telemetry](pkg/telemetry/README.md)

Модуль для настройки отправки трейсов, а также хелперов для сбора метрик.
Включён в шаблон проекта

#### [Server](pkg/server/README.md)

Модуль для запуска сервисов в http и grpc.
По умолчанию включены сервисы для обработки запросов информации о приложении и метрик.
Включён в шаблон проекта

#### [RateLimiter](pkg/ratelimiter/README.md)

Модуль для работы с лимитером на основе кеша и redis. Подключить можно через `fx`

#### 5.2.1 Состав модулей Connections

Набор модулей для работы с различными базами данных и сервисами, postgres, redis cluster, kafka, grpc.

Каждый модуль подключется через `fx`

##### [GRPC Clients](pkg/connections/grpccli/README.md)

Модуль для работы с grpc клиентами, реализованный через модель пулов, где для каждого коннекта свой конфиг и объект grpc клиента

##### [Kafka](pkg/connections/kafka/README.md)

Модуль для работы с kafka, реализованныей через модель пулов, где для каждого коннекта свой конфиг

##### [Redis](pkg/connections/rediscli/README.md)

Модуль для работы с redis, реализованный через модель пулов, где для каждого коннекта свой конфиг и объект universal redis клиента

##### [Postgres](pkg/connections/postgres/README.md)

Модуль для работы с postgres, реализованный через модель пулов, где для каждого коннекта свой конфиг и объект postgres клиента
